# implement

小さく実装して、検証して、PR 本文まで整える。

## ワークフロー

### 1. 着手前の宣言
```markdown
## Goal
[何を達成するか]

## Done
- [ ] 完了条件

## Touched files
- `path/to/file.ts`
```

### 2. 実装の原則

**最小変更**: 依頼された範囲のみ。無関係な整形・リネームはしない。

**既存コードの尊重**: 命名規則・パターン・lint設定に従う。

**セキュリティ**: ユーザー入力のバリデーション、インジェクション防止、secrets をハードコードしない。

### 3. テスト追加

変更箇所にはテストを追加:
- 正常系
- 異常系
- 境界値

### 4. 検証

```bash
npm run lint  # or yarn lint
npm test      # or pytest
npm run build # 必要に応じて
```

### 5. PR本文

`docs/agent/PR.md` 形式で出力:
```markdown
## 概要
### What
[変更内容]

### Why
[背景・目的]

## 変更点
- `path/to/file.ts`: [内容]

## 動作確認
- [ ] テスト通過
- [ ] lint 通過
```

## 2回続けて失敗したら

```markdown
## 状況
- 何が起きているか
- 原因仮説
- 次に試すこと
```

## このスキルでできないこと
- タスク分解（`/orchestrator` を使用）
- バグの根本原因調査（`/debug` を使用）
