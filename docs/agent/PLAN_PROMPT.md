# Plan プロンプト

Issue #{{ISSUE_NUMBER}} の依頼内容を分析し、実装プランを作成して Issue コメントに投稿してください。

## 手順

1. `gh issue view {{ISSUE_NUMBER}}` で Issue の詳細を確認する
2. コードベースを調査して影響範囲・変更対象ファイルを把握する
3. 以下のフォーマットでプランを作成し、`gh issue comment {{ISSUE_NUMBER}} --body "..."` で投稿する

## プランのフォーマット

```
## 実装プラン

### 概要
（何をするか・なぜするか）

### 変更対象ファイル
- `path/to/file` - 変更内容

### 実装ステップ
- [ ] ステップ1
- [ ] ステップ2

### 懸念点・確認事項
（あれば記載。なければ省略）

---
実装を開始するには `@claude implement` とコメントしてください。
```

**重要**: コードの実装・ファイルの書き込み・ブランチ操作は行わないこと。プランの投稿のみ行う。
